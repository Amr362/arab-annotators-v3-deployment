# ✅ المرحلة الثالثة — Manager Role + API Redesign مكتملة

--- 

## ملخص ما تم إنجازه:

تم الانتهاء بنجاح من المرحلة الثالثة لتطوير منصة Arab Annotators، والتي ركزت على إضافة دور المدير (Manager Role) وإعادة تصميم واجهة برمجة التطبيقات (API) لدعم الميزات الجديدة. تضمنت هذه المرحلة تحديثات جوهرية على كل من الواجهة الخلفية (Backend) والواجهة الأمامية (Frontend) لتعزيز إدارة المشاريع والمستخدمين والجودة.

### الملفات الجديدة:

تم إضافة ثلاثة ملفات جديدة رئيسية لدعم وظائف المدير:

| الملف | الوصف |
|---|---|
| `server/managerRouter.ts` | يحتوي على 587 سطرًا من التعليمات البرمجية و21 إجراءً (procedure) لإدارة المشاريع والفريق والجودة بشكل شامل. |
| `server/skipRateLimiter.ts` | يطبق حدًا للتخطي (Skip Rate Limiting) بواقع 3 تخطيات لكل ساعة لكل مشروع، مع آلية عد تنازلي (countdown) لتتبع التخطيات المتبقية. |
| `client/src/pages/ManagerDashboard.tsx` | يمثل لوحة تحكم المدير بواقع 718 سطرًا من التعليمات البرمجية، ويحتوي على 5 تبويبات (tabs) كاملة توفر واجهة متكاملة للمدير. |

### التحديثات الجوهرية:

تم إجراء تحديثات مهمة على الملفات الموجودة لدمج دور المدير والميزات الجديدة:

| الملف | التغيير |
|---|---|
| `server/_core/trpc.ts` | إضافة إجراءات جديدة خاصة بالأدوار: `managerProcedure`، `qaProcedure`، و`taskerProcedure` لضمان التحكم الدقيق في الوصول بناءً على الصلاحيات. |
| `server/routers.ts` | ربط `managerRouter` الجديد ودمج منطق حد التخطي (skip rate-limiting) ضمن توجيهات API. |
| `App.tsx` + `Login.tsx` | تحديث مسارات التطبيق لتشمل مسار `/manager`، وتعديل صفحة تسجيل الدخول لتوجيه المستخدمين ذوي دور المدير تلقائيًا إلى لوحة تحكم المدير. |
| `TaskerDashboard.tsx` | إضافة عداد التخطيات (skip quota) لعرض عدد التخطيات المتبقية للموسم، وتفعيل استدعاء `startTask` تلقائيًا عند فتح المهمة لبدء تتبع وقت العمل. |
| `Admin.tsx` | إضافة خيار 
"مدير مشاريع" في واجهة إدارة المستخدمين، مما يتيح للمسؤولين تعيين هذا الدور للمستخدمين. |

### ميزات لوحة تحكم المدير:

توفر لوحة تحكم المدير الجديدة واجهة شاملة لإدارة المشاريع والفريق. تشمل هذه الواجهة قسمًا لـ **نظرة عامة (Overview)** يعرض مؤشرات الأداء الرئيسية (KPIs)، ومخططًا دائريًا لتوزيع المهام، ولوحة صدارة تعرض أفضل الموسمين. كما تتضمن **قائمة المراجعة (Review List)** التي تتيح للمدير قبول المهام المنجزة، أو تعديلها، أو رفضها مع إمكانية إضافة ملاحظات توجيهية للموسمين. لإدارة الفريق، يوفر قسم **الفريق (Team Management)** أدوات لتعيين أعضاء الفريق، وتحديد مستويات مهاراتهم، ورفع الإيقاف عن الحسابات المعلقة. أما قسم **الدُّفعات (Batches)** فيسمح بإنشاء دفعات جديدة من المهام مع إمكانية تحديد نسبة مهام اختبار الجودة (HP) ونسبة المراجعة (QA) بشكل مخصص. أخيرًا، يتيح قسم **تحليل التوافق (IAA)** للمدير تقييم مدى التوافق بين الموسمين باستخدام مقاييس Fleiss' κ و Cohen's κ، مع توفير رسوم بيانية توضيحية لتحليل النتائج.

تمثل هذه التحديثات خطوة مهمة نحو تحسين كفاءة إدارة المشاريع في منصة Arab Annotators، وتوفير أدوات متقدمة لضمان جودة البيانات المصنفة.
